WTF is git, anyway?

Git is magical!
It is also infuriating.

But!
It will totally make your life easier.

git is version control

  • version control = source control = revision control
  • A system for tracking changes.
  • There are lots of different systems.
  • We’re using git because it’s better (trust me).

Why do we use it?

Let’s install it!

  • brew install git on Mac
  • git-scm.com/downloads everywhere else
  • help.github.com/articles/set-up-git

Oh, right!
We should probably install WordPress somewhere, too.

Install WordPress somewhere

  • You should know how to do this.
  • (wordpress.org)
  • Install it on one server, set up your .gitignore, commit all files, push!
  • And then pull on the other.

Gotcha numero dos
Server-specific setup required!

Don’t ignore your .gitignore!

  • Keep machine-specific files out of git (ex: wp-config.php).
  • Restrictive hosting: keep sensitive files out of git.
  • Focus on theme files.
  • Flexible hosting: put everything in git.
  • Production database sync, WordPress core files, plugins.

Database syncing

  • Run a few shell commands to do the heavy lifting.
  • Keep content, core, plugin, and user data in-sync with files.
  • gist.github.com/sarahsemark/5889465

Seriously, why bother?

  • You change settings in two different admin panels.
  • You always need data from production site.
  • You use PHPMyAdmin to sync database changes (that’s so 2009.)

Huh?

Other gotchas

  • Want to remove a file from git but keep on your machine?
    git rm --cached and add to .gitignore
  • Deleted a bunch of files?
    git add -u removes all deleted files from git
  • Make sure your $table_prefix is the same everywhere!

Security

  • Keep your .git folder out of public_html
  • Don’t expose scripts, passwords, or database sync info in public_html
  • Basically, remember that public_html is public
